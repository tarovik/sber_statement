# sber_statement

[![Tests](https://github.com/YOUR_USER/sber_statement/actions/workflows/test.yml/badge.svg)](https://github.com/YOUR_USER/sber_statement/actions/workflows/test.yml)

Парсинг выписки СберБанка из PDF в CSV.

> ⚠️ Замените `YOUR_USER` в ссылке бейджа на ваш GitHub username после создания репозитория.

Извлекает все операции (зачисления, списания, мультивалютные транзакции) из слабоструктурированного PDF-формата СберБанка и сохраняет в плоский CSV-файл для дальнейшего анализа.

## Возможности

- Извлечение всех операций из PDF-выписки СберБанка (дебетовые карты)
- Распознавание мультивалютных транзакций (RSD, EUR, USD, GBP, CHF, JPY и др.)
- Поддержка отрицательных сумм в валюте (возвраты/refund)
- Поддержка JPY и других валют без десятичных знаков (целые числа)
- Автоматическая проверка баланса: начальное сальдо + операции = конечное сальдо
- Поддержка русского формата чисел (пробел как разделитель тысяч, запятая как десятичный разделитель)
- CSV с разделителем `;` (точка с запятой) — совместимость с Excel
- Два режима: точка или запятая как десятичный разделитель (`--decimal-comma`)
- Подробный лог-файл с результатами проверки баланса

## Запуск

Скрипт требует **Python 3.9+** и библиотеку `pdfplumber`.

### Windows (cmd или PowerShell)

```cmd
pip install pdfplumber
python parse.py Выписка.pdf
```

### macOS / Linux

```bash
pip3 install pdfplumber
python3 parse.py "Выписка.pdf"
```

### Результат

- `Выписка.csv` — данные в плоском формате (открывается в Excel)
- `Выписка.log` — отчёт о проверке баланса

### Опции

```bash
# Указать имя выходного файла
python parse.py input.pdf output.csv

# Использовать запятую как десятичный разделитель (для русского Excel)
python parse.py input.pdf --decimal-comma
```

## Формат CSV

| Колонка | Описание | Пример |
|---|---|---|
| `date_operation` | Дата операции | `2023-01-13` |
| `operation_time` | Время операции | `08:33` |
| `date_processing` | Дата обработки | `2023-01-13` |
| `auth_code` | Код авторизации | `258766` |
| `category` | Категория операции | `Прочие операции` |
| `description` | Описание операции | `Заработная плата. Операция по карте ****8340` |
| `amount_rub` | Сумма в рублях | `35910.00` |
| `balance` | Остаток после операции | `58494.52` |
| `card_account` | Номер карты/счёта | `****8340` |
| `currency_code` | Код валюты (если есть) | `RSD`, `EUR`, `JPY` |
| `amount_currency` | Сумма в валюте | `2331.00` |

## Как это работает

1. PDF читается через `pdfplumber`, текст извлекается постранично
2. Заголовок выписки парсится (владелец счёта, номер счёта, период, остатки, итоги)
3. Транзакции парсятся: каждая состоит из двух строк (дата/время/категория/сумма/остаток + дата обработки/код/описание)
4. Мультивалютные операции определяются по 3-буквенному коду в конце описания
5. Баланс проверяется: `начальное сальдо + зачисления - списания = конечное сальдо`
6. Результат сохраняется в CSV и лог-файл
